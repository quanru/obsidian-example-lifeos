- アーカイブとは、他の3つのカテゴリから非アクティブになった項目のことである。

アーカイブとは、現在は積極的に関与・維持していないが、将来的に役立つ可能性のある情報を指す。

例：

- 旧プロジェクトのファイル
- 期限切れのリソース
- 未使用のノート

アーカイブ内の情報には、次のような特徴があります：

1. 現在はアクティブではない
2. 将来の参照として価値がある可能性がある
3. 頻繁な参照やメンテナンスを必要としない
4. 検索可能な状態を維持している

情報のアーカイブとは直接削除することではなく、更新やメンテナンスへの積極的な関与を停止するようにステータスを変更することです。

例：

プロジェクト：完成した原稿

リソース：視聴済みだが未整理の動画

エリア：以前の関心事で、現在は注力していないもの

その他の種類の情報も分類してアーカイブできます。

将来必要になった場合は、検索やその他の方法で取り出すことができます。

受動的な削除と比べて、アーカイブには次のメリットがあります：

1. 検索の手間を減らす
2. 情報の損失を防ぐ
3. 管理しやすくなる

そのため、アーカイブはP.A.R.Aシステムの重要なモジュールです。
