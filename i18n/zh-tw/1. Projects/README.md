- 專案是一系列與目標掛鉤、有截止期限的任務集合。

專案是指有明確目標與截止期限的任務或活動。

例如：

- 寫一本書（3 個月）
- 開發一款應用程式（6 個月）
- 設計一個網站（2 週）
- 規劃週末烤肉活動（1 天）

專案相較於領域更為具體，是完成任務的最小執行單位。

專案具有以下特徵：

1. 明確的開始時間與預期完成時間
2. 具體的目標與預期可交付成果
3. 可拆解為一系列具有先後關係的任務
4. 完成後能產生具體的價值或影響

專案不一定要非常複雜，日常生活中的小目標也可以視為一個專案。

在領域的層面上，專案有助於實現某個特定領域內的願景。

例如：

領域：前端開發

相關專案：

- 建立 React 專案架構（1 週）
- 實作登入／註冊功能（3 天）
- 重構資料流（2 天）

透過持續完成這些有助於該領域的專案，可以在該領域中不斷精進與成長。

專案與領域相輔相成，是 PARA 系統中重要的二元對立關係。
